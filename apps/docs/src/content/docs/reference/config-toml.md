# `.deviate/config.toml` Schema

Per-project DeviaTDD configuration loaded from `<workdir>/.deviate/config.toml`. Validated by the `DeviateConfig` Pydantic model in `src/deviate/state/config.py`; emitted by `deviate setup` via `_dict_to_toml` in `src/deviate/cli/__init__.py`.

## Top-level scalars

| Field | Type | Default | Description |
|---|---|---|---|
| `profile` | `string` | `"default"` | Preset config-group name; documented values are `"default"`, `"full"`, `"fast"`, `"secure"` (freeform `str`, not enum-validated) |
| `timeout_seconds` | `int` | `300` | CLI inactivity timeout in seconds; must be greater than 0 (`gt=0`) |
| `agent_export_mode` | `enum` | `"local"` | Where agent slash commands are exported: `"local"` writes to `<workdir>/.claude/`, `"global"` writes to `~/.claude/` |
| `use_libref` | `bool` | `false` | Enable the libref CLI for offline documentation lookups during agent invocations |
| `graphite` | `bool` | `false` | Enable Graphite CLI integration for stacked changes (`gt create` / `gt submit` / `gt sync`) |

## `[agent]` table

| Field | Type | Default | Description |
|---|---|---|---|
| `backend` | `enum` | `"opencode"` | Agent backend that meso/micro layers invoke; one of `"opencode"`, `"claude"`, `"droid"`, `"pi"` |
| `timeout` | `int` | `600` | Agent invocation timeout in seconds; must be greater than 0 (`gt=0`) |
| `pi_rpc` | `bool` | `false` | Opt-in RPC mode for Pi; spawns `pi --mode rpc --no-session` instead of `pi -p` |

Note: the user-facing prompt maps `factory` → `droid` (Factory Droid IDE runs the `droid` binary), but `factory` is not a valid stored `backend` value — the literal in the file is always `droid`. Mapping lives in `src/deviate/cli/__init__.py::AGENT_TO_BACKEND`.

## `[models]` table

Per-phase model overrides; keys are phase names (case-insensitive), values are backend model IDs. Resolution order via `resolve_phase_model` in `src/deviate/state/config.py`:

1. Phase-specific key (e.g., `judge`, `plan`, `red`)
2. `default` key
3. `None` — backend falls back to its native default

| Field | Type | Default | Description |
|---|---|---|---|
| `<phase>` | `string` | `null` | Model ID override for one phase; valid phase names include `idle`, `explore`, `research`, `prd`, `shard`, `specify`, `plan`, `tasks`, `red`, `green`, `yellow`, `judge`, `refactor`, `e2e`, `execute`, `hotfix` |
| `default` | `string` | `null` | Fallback model ID applied to any phase without an explicit override |

The `opencode` and `droid` backends accept the resolved value via `--model <id>`; the `claude` backend ignores it silently.

## Emit order

`deviate setup` writes the file via `_dict_to_toml`, which guarantees scalar top-level keys precede table headers (`[agent]`, `[models]`). TOML grammar absorbs bare keys after a `[table]` header into that table, so user edits to top-level scalars stay at root scope only when the file is generated by `deviate setup` (or hand-edited while preserving scalar-first ordering). Emission order:

1. Scalars (in source order): `profile`, `timeout_seconds`, `agent_export_mode`, `use_libref`, `graphite`
2. Tables (in source order): `[agent]`, `[models]`

Example:

```toml
# Preset config group: "default", "full", "fast", or "secure"
profile = "default"
# CLI inactivity timeout in seconds (must be > 0)
timeout_seconds = 300
```

Example:

```toml
# Agent backend configuration
[agent]
backend = "opencode"
timeout = 600
pi_rpc = false
```

## Validation

| Constraint | Behaviour |
|---|---|
| Unknown top-level key | `ValidationError` (`extra = "forbid"` on `DeviateConfig`) |
| `timeout_seconds <= 0` | `ValidationError` (`gt=0`) |
| Unknown `agent.backend` | `ValidationError` (`Literal["opencode", "claude", "droid", "pi"]`) |
| Unknown `agent_export_mode` | `ValidationError` (`Literal["local", "global"]`) |
| Missing file | Loader returns `None`; `resolve_phase_model` and `resolve_graphite_config` fall back to built-in defaults |
