# Starter Config

Three files written by `deviate setup` to a fresh DeviaTDD workspace: `.deviate/config.toml` (default config), project-root `.gitignore` (agent-command exclusions), and project-root `.gitattributes` (append-only ledger union-merge rules).

## `.deviate/config.toml`

The starter config is generated by `_dict_to_toml(DeviateConfig(...).model_dump(), comments=_CONFIG_TOML_COMMENTS)` in `src/deviate/cli/__init__.py` and stamped by `_scaffold_dotfiles`. All fields below are the `DeviateConfig` defaults (no overrides); re-running `deviate setup` does not edit an existing file unless `--graphite` or `--libref` is passed (then a surgical key merge runs).

### Top-level scalars

| Field | Type | Default | Description |
|---|---|---|---|
| `profile` | `string` | `"default"` | Preset config-group name; documented values include `"default"`, `"full"`, `"fast"`, `"secure"` |
| `timeout_seconds` | `int` | `300` | CLI inactivity timeout in seconds; must be greater than 0 (`gt=0`) |
| `agent_export_mode` | `enum` | `"local"` | Where agent commands are exported: `"local"` writes to `<workdir>/.claude/`, `"global"` writes to `~/.claude/` |
| `use_libref` | `bool` | `false` | Enable the libref CLI for offline documentation lookups during agent invocations |
| `graphite` | `bool` | `false` | Enable Graphite CLI integration for stacked changes (`gt create` / `gt submit` / `gt sync`) |

### `[agent]` table

| Field | Type | Default | Description |
|---|---|---|---|
| `backend` | `enum` | `"opencode"` | Agent backend that meso/micro layers invoke; one of `"opencode"`, `"claude"`, `"droid"`, `"pi"` |
| `timeout` | `int` | `600` | Agent invocation timeout in seconds; must be greater than 0 (`gt=0`) |
| `pi_rpc` | `bool` | `false` | Opt-in RPC mode for Pi; spawns `pi --mode rpc --no-session` instead of `pi -p` |

Note: `[models]` is not emitted in the starter file because `_dict_to_toml` skips empty tables (the `DeviateConfig.models` default is `{}`). Once you add a phase override — e.g. `default = "opencode/deepseek-v4-flash"` — the `[models]` table appears in the file. See [`.deviate/config.toml` Schema](/reference/config-toml) for the full schema, validation rules, and resolver order.

### Starter TOML

Example:

```toml
# Preset config group: "default", "full", "fast", or "secure"
profile = "default"
# CLI inactivity timeout in seconds (must be > 0)
timeout_seconds = 300
```

Example:

```toml
# Agent backend configuration

[agent]
backend = "opencode"
timeout = 600
pi_rpc = false
```

## Project-root `.gitignore`

Generated by `_ensure_root_gitignore` in `src/deviate/cli/__init__.py`. Idempotent: re-running `deviate setup` does not duplicate entries and never rewrites user-authored lines. Created when the project-root `.gitignore` does not exist; otherwise merged.

| Pattern | Type | Description |
|---|---|---|
| `*/commands/deviate-*.md` | gitignore-pattern | Excludes DeviaTDD slash commands under any agent's `commands/` install |
| `*/commands/tome-*.md` | gitignore-pattern | Excludes Tome Subsystem slash commands under any agent's `commands/` install |
| `*/prompts/deviate-*.md` | gitignore-pattern | Excludes DeviaTDD slash commands under `.pi/prompts/` (Pi platform convention) |
| `*/prompts/tome-*.md` | gitignore-pattern | Excludes Tome Subsystem slash commands under `.pi/prompts/` |

The `*/commands/` and `*/prompts/` prefixes scope each pattern to exactly one directory level — broader globbing (`**/deviate-*.md`) would silently ignore source files at `src/deviate/prompts/commands/deviate-*.md` and break `deviate setup` in the deviatdd repo itself. The four patterns cover every supported agent (`.claude/`, `.opencode/`, `.factory/`, `.pi/`) and any future agent that follows the same `<dir>/commands/` or `<dir>/prompts/` flat-file convention.

## Project-root `.gitattributes`

Generated by `_ensure_root_gitattributes` from the `DEVIATE_GITATTRIBUTES_SEED` constant in `src/deviate/cli/__init__.py`. Without these rules, concurrent `deviate shard` runs on feature branches produce line-level conflicts in `specs/issues.jsonl` at merge time. `merge=union` is git's built-in line-wise union driver — it keeps every unique line across all branches and emits no conflict markers.

| Attribute | Value | Description |
|---|---|---|
| `specs/issues.jsonl` | `merge=union` | Union-merge the macro-layer append-only issue ledger |
| `specs/**/tasks.jsonl` | `merge=union` | Union-merge the meso-layer per-epic task ledgers |

The seeder strips the comment block from `DEVIATE_GITATTRIBUTES_SEED` and idempotently merges only the two attribute lines — user-authored entries are preserved, re-runs never duplicate rules.
