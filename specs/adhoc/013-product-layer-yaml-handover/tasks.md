## Implementation Strategy
**Execution Order**:
1. Phase 1 (`TSK-013-01`) → Phase 2 (`TSK-013-02`) and Phase 3 (`TSK-013-03`) in parallel (logically independent)

The three tasks are mutually independent at the implementation layer — Phase 1 appends SKILL.md markers, Phase 2 verifies the path-shape contract, Phase 3 documents the convention. Phase 2's tests pass against the existing `handover_path()` without waiting for Phase 1's SKILL.md appends (the path-shape contract is consumed unchanged). Phase 3's docs reference the path shape established by Phase 2's tests and used by Phase 1's SKILL.md appends, but the docs are pure markdown and do not require code-level coordination.

**Critical Dependency Chains**:
- None — all three tasks are independent and may be merged in any order. The Implementation Strategy lists them in narrative dependency order (Phase 1 writes the marker, Phase 2 verifies the path surface, Phase 3 documents the convention) but the executor MAY parallelize Phase 2 + Phase 3 if the test runner and the markdown editor are decoupled.
