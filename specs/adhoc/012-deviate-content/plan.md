## Data Flow Analysis
1. **Capture (FLOW-11)**: A DeviaTDD phase (e.g. `judge`) completes → skill actor reads its modified `SKILL.md` (now with the one-sentence Write instruction) → actor emits YAML manifest on stdout AND/OR via the Write tool to `.deviate/feat/<epic>/<issue>/[<task>/]<phase>.yaml` → runner's `persist_handover()` validates the resolved path is rooted under `.deviate/feat/` (raises `PathTraversalError` otherwise) → writes the YAML via `yaml.safe_dump` only if the file is absent (idempotent) → returns the `Path` → phase post-handler exits. `.deviate/feat/**` is gitignored; no commit, no ledger append.
2. **Storage**: YAMLs are persisted as files on disk under `.deviate/feat/<epic_slug>/<issue_id>/[<task_id>/]<phase>.yaml`. Each YAML is a `phase`, `status`, `files`, and (optionally) `narrative_anchor:` block whose field names are phase-specific per `specs/plans/deviate-content.md:52-67`. No write-side Pydantic model is involved — the file content is the durable artifact.
3. **Read-back (FLOW-12 input)**: `load_handover_records(window: str | None) -> list[HandoverRecord]` globs `.deviate/feat/**/*.yaml`, parses each via `yaml.safe_load` (skipping malformed entries with a stderr warning), and returns chronologically ordered `HandoverRecord` Pydantic models. The `HandoverRecord` is read-side only — synthesized for the synthesis layer, never written.
4. **Synthesis (FLOW-12)**: Developer runs `deviate content --format blog --slug my-post [--window EPIC-X] [--archive EPIC-X]` → CLI calls `load_handover_records(window)` to gather matching YAMLs → for each record, extract `narrative_anchor` keyed by phase-specific field names → render via the format template from `src/deviate/prompts/content/` (using `str.format()` substitution, NO Jinja2) → write draft to `.deviate/content-drafts/<format>/<slug>.md` (gitignored) → developer reviews and publishes manually.
5. **Archive (FLOW-12 side-channel)**: When `--archive EPIC-X` is invoked, the synthesis path is replaced with a tarball write: every YAML under `.deviate/feat/EPIC-X/**` is added to `specs/_archives/EPIC-X-narrative.tar.gz` via Python stdlib `tarfile.open(mode="w:gz")`. The tarball is the only committed-by-default artifact of the Content Capture subsystem.

## Risk Assessment
| Risk | Impact | Likelihood | Mitigation |
|------|--------|------------|------------|
| LLM actor forgets the Write call on the manual path | Medium | Medium | `persist_handover()` checks `pathlib.Path.exists()` for the canonical file; on CLI path the runner parses stdout and writes unconditionally, so the failure mode is manual-path-only. Detected in `test_persist_manual_path`. |
| LLM actor writes malformed YAML | Low | Medium | `yaml.safe_load` in `load_handover_records()` skips with stderr warning and continues. Detected in `test_load_records` (malformed fixture). |
| LLM actor writes to wrong path on CLI path | Medium | Low | Post-handler validates resolved path is rooted under `.deviate/feat/` via `Path.resolve(strict=False)`. Path-traversal guard raises `PathTraversalError` before any write. Detected in `test_path_resolution` (traversal fixture). |
| Path traversal via `task_id` or `epic_slug` containing `..` | High | Low | Same guard as above — `Path.resolve(strict=False)` and root-prefix check. Detected via `test_persist_manual_path` (traversal input). |
| Re-running `persist_handover()` overwrites legitimate diffs | Medium | Low | Idempotent overwrite-or-skip: file existence check + early return. Divergent content from a re-run is a real failure surfaced by `git log` diff (out of YAML layer scope). Detected in `test_idempotent_write`. |
| 15-skill append accidentally duplicates instruction on re-run | Low | Medium | Idempotency guard: detect literal "handover_path()" already present, skip. Detected via the `test_skill_prompts.py` re-run scenario. |
| Cross-cutting touch on 15 skills breaks an existing skill-validation test | High | Low | Each append is a single sentence at the END of the terminal-contract block — no body content is rewritten. Re-run the full `mise run test` suite to catch regressions. The 15 appends are mechanical and identical. |
| `tarfile` import on systems without gzip support | Low | Low | `tarfile` is Python stdlib; `gzip` is stdlib. No external dependency. Detected via the `test_archive_flag` smoke. |
| `yaml.safe_load` performance on a 200-YAML repo | Low | Low | Sequential load is fast (≤ 100ms for 200 records per Performance Constraint). No optimization needed in v1. |
